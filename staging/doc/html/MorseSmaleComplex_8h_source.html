29 /*
30  * Morse-Smale complex developer quick guide:
31  *
32  * What is the architecture?
33  * -------------------------
34  * The DiscreteGradient class contains the basic structures to define and build
35  * a discrete gradient. It also has several functions that decrease the number
36  * of unpaired cells as parallel post-processing steps. Even more work is done
37  * on the gradient with a sequential simplification step. Finally, it is able
38  * to build critical points and vpaths.
39  * Its files are:
40  * DiscreteGradient.cpp
41  * DiscreteGradient.h
42  *
43  * The AbstractMorseSmaleComplex class contains whatever is common
44  * between the MorseSmaleComplex2D class and the MorseSmaleComplex3D class
45  * e.g. parameters, configuration functions, input and output data pointers.
46  * In particular, it contains a DiscreteGradient attribute and a Triangulation
47  * attribute.
48  * Its files are:
49  * AbstractMorseSmaleComplex.cpp
50  * AbstractMorseSmaleComplex.h
51  *
52  * The MorseSmaleComplex2D class is derived from the AbstractMorseSmaleComplex
53  * class. It is specialized in building the Morse-Smale complex on 2D
54  * triangulations. This class uses the DiscreteGradient attribute to build a
55  * valid discrete gradient before building the MSC outputs itself i.e.
56  * critical points, 1-separatrices, segmentation.
57  * Its files are:
58  * MorseSmaleComplex2D.cpp
59  * MorseSmaleComplex2D.h
60  *
61  * The MorseSmaleComplex3D class is derived from the AbstractMorseSmaleComplex
62  * class. It does the same job as the MorseSmaleComplex2D class but on 3D
63  * triangulations. Note that this class has a function
64  * computePersistencePairs() to get the saddle-saddle pairs of the data.
65  * It adds the saddle-connectors to the 1-separatrices and adds another output
66  * for 2-separatrices.
67  * Its files are:
68  * MorseSmaleComplex3D.cpp
69  * MorseSmaleComplex3D.h
70  *
71  * The MorseSmaleComplex class is derived from the AbstractMorseSmaleComplex
72  * class. It is a convenience class that detects the dimensionality of the data
73  * and uses the correct implementation of the Morse-Smale complex computation
74  * (between MorseSmaleComplex2D and MorseSmaleComplex3D).
75  * Its files are:
76  * MorseSmaleComplex.cpp
77  * MorseSmaleComplex.h
78  *
79  * How to build the gradient?
80  * --------------------------
81  * Everything that concerns the gradient is in the DiscreteGradient class.
82  * In order to build a valid discrete gradient you need to first set the data
83  * pointers to the input scalar field, offset field and triangulation. You need
84  * to set also the data pointers to the output critical points, 1-separatrices,
85  * 2-separatrices and segmentation. Additional parameters can be configured
86  * like an iteration threshold, options to have PL-Compliant extrema and
87  * saddles, an option to enable collecting of persistence pairs or
88  * post-processing of the saddle-connectors. Note that they all have default
89  * values that correspond to a standard scenario. Like any other TTK
90  * module, the level of debug and the number of threads can be adjusted to suit
91  * your needs.
92  * Once all the parameters and data pointers are set, the function
93  * buildGradient() builds the discrete gradient. As a substantial number of
94  * unpaired cells is expected, it is strongly recommended to use after this
95  * function the function buildGradient2() and after that the buildGradient3()
96  * function if the input dataset is in the 3D domain.
97  * Finally, you can apply reverseGradient() to auto-detect the PL critical
98  * points and impose that the gradient is PL-Compliant (except on the
99  * boundary). See the related publication "The Topology ToolKit" for further
100  * details.
101  * Examples of such usage of the DiscreteGradient class can be found in
102  * the execute() function of the MorseSmaleComplex2D class and
103  * MorseSmaleComplex3D class as these classes need to compute a discrete
104  * gradient to build their own outputs.
105  *
106  * Where is the simplification algorithm?
107  * --------------------------------------
108  * The main steps of the gradient simplification algorithm are stored in the
109  * reverseGradient() function in the DiscreteGradient class. More informations
110  * can be found in each simplify-like function as the process is slightly
111  * different depending on the index of the critical points involved:
112  * simplifySaddleMaximumConnections() for reversal of (saddle,...,maximum)
113  * vpaths.
114  * simplifySaddleSaddleConnections1() for reversal of
115  * (2-saddle,...,1-saddle) vpaths.
116  * simplifySaddleSaddleConnections2() for reversal of
117  * (1-saddle,...,2-saddle) vpaths.
118  *
119  * How to add a scalar field on any output geometry?
120  * -------------------------------------------------
121  * First, go to AbstractMorseSmaleComplex.h and add the void* pointer to a STL
122  * container (e.g. vector) as a class attribute. In the code, the attributes
123  * of the same output are grouped together and are prefixed by its name.
124  * So for example, the outputSeparatrices1_points_smoothingMask_ variable
125  * represents the smoothingMask scalar field that is associated to the points
126  * of the 1-separatrices of the Morse-Smale complex. As you just added a new
127  * attribute, you need to update the setter corresponding to the output to
128  * add this new element:
129  * The setters available are:
130  * setOutputCriticalPoints()
131  * setOutputSeparatrices1()
132  * setOutputSeparatrices2()
133  * setOutputMorseComplexes()
134  * As you added a new attribute in the class, you need to give it a default
135  * value in the constructor in AbstractMorseSmaleComplex.cpp (typically nullptr
136  * for a pointer).
137  * Now, you need to overload the same setter than previously but in
138  * MorseSmaleComplex.h this time in order to propagate the new field
139  * information to the concrete implementations. As the MorseSmaleComplex2D and
140  * MorseSmaleComplex3D classes are both derived from AbstractMorseSmaleComplex
141  * they already have the updated version of the function.
142  * Now that you have access to the data pointer inside the actual
143  * implementation you can cast it from void* to dataType* and modify it to
144  * your convenience.
145  *
146  * What part of the code is parallel?
147  * ----------------------------------
148  * From a global point a view, the part of the code that builds the gradient
149  * as well as the two post-processing steps are accelerated by OpenMP (if
150  * available).
151  * The gradient simplification step is mostly sequential except the
152  * initialization of internal structures in the initialize-like functions which
153  * is done is parallel with OpenMP.
154  * Then, each output of the Morse-Smale complex is computed in parallel with
155  * OpenMP except the 2-separatrices of a 3D dataset that require heavy
156  * synchronisation.
157  * Finally, the generation of the geometry for the visualization is done
158  * sequentially.
159  * Complete list:
160  * In the DiscreteGradient class:
161  * buildGradient()
162  * buildGradient2()
163  * buildGradient3()
164  * initializeSaddleMaximumConnections()
165  * initializeSaddleSaddleConnections1()
166  * initializeSaddleSaddleConnections2()
167  * In the MorseSmaleComplex2D class:
168  * getSeparatrices1()
169  * setAscendingSegmentation()
170  * setDescendingSegmentation()
171  * In the MorseSmaleComplex3D class:
172  * getSeparatrices1()
173  * getAscendingSeparatrices2()
174  * getDescendingSeparatrices2()
175  * setAscendingSegmentation()
176  * setDescendingSegmentation()
177  */
